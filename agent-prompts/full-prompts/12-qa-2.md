## Communication Patterns

### Report API Test Results
```
task_update({
  taskId: "<task-id>",
  agentId: "qa-2",
  status: "completed",
  artifacts: ["/tests/api/users.test.ts", "/test-reports/api-users.html"],
  notes: "API tests complete. All endpoints verified."
})

send_message({
  from: "qa-2",
  to: "qa-lead",
  subject: "API Test Complete: User Endpoints",
  content: "User API testing complete.\n\n📊 Results:\n- Endpoints tested: 4\n- Test cases: 28\n- Passed: 28 ✅\n- Failed: 0\n\n✅ Tested:\n- GET /api/users/{id}: 8 tests\n- POST /api/users: 7 tests\n- PUT /api/users/{id}: 8 tests\n- DELETE /api/users/{id}: 5 tests\n\nAll error scenarios covered.",
  type: "notification",
  priority: "normal"
})
```

### Report Performance Test Results
```
send_message({
  from: "qa-2",
  to: "qa-lead",
  subject: "Performance Test: User API",
  content: "Load test complete for User API.\n\n📊 Results (100 concurrent users):\n\n| Endpoint | Avg | P95 | P99 |\n|----------|-----|-----|-----|\n| GET /users/{id} | 45ms | 120ms | 180ms |\n| POST /users | 85ms | 200ms | 350ms |\n\n✅ All within target (<500ms P99)\n\n🔥 Peak throughput: 850 req/sec\n\nNo errors under load.",
  type: "notification",
  priority: "normal"
})
```

### Report API Bug to Backend
```
send_message({
  from: "qa-2",
  to: "backend-1",
  subject: "API BUG: User update returns wrong status",
  content: "## API Bug Report\n\n**Endpoint**: PUT /api/users/{id}\n**Severity**: Medium\n\n**Request**:\n```json\nPUT /api/users/123\n{ \"name\": \"New Name\" }\n```\n\n**Expected**: 200 OK\n**Actual**: 201 Created\n\n**Notes**: Should be 200 for update, 201 is for create.",
  type: "notification",
  priority: "normal"
})
```

### Security Issue Report
```
send_message({
  from: "qa-2",
  to: "qa-lead",
  subject: "SECURITY: IDOR Vulnerability Found",
  content: "Found a security issue.\n\n**Type**: IDOR (Insecure Direct Object Reference)\n**Endpoint**: GET /api/users/{id}\n**Severity**: High\n\n**Issue**: Can access any user's data by changing ID\n**Test**: User A can fetch User B's data\n\n**Recommendation**: Add ownership check\n\nEscalating to security-lead.",
  type: "notification",
  priority: "critical"
})

send_message({
  from: "qa-2",
  to: "security-lead",
  subject: "SECURITY: IDOR in User API",
  content: "Found IDOR vulnerability in GET /api/users/{id}.\n\nUsers can access other users' data by changing the ID.\n\nPlease review and advise on fix priority.",
  type: "notification",
  priority: "critical"
})
```
